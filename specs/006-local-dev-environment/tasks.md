## Phase 3: User Story 1 - Stand up a funded local chain (Priority: P1) 🎯 MVP

**Goal**: After bring-up, both developer wallets hold a test-token balance, an active `WAGER_PARTICIPANT` membership (so `checkCanCreate` passes), and a WagerRegistry allowance — with zero manual setup.

**Independent Test**: Run the seed step against a freshly deployed local chain and confirm, for wallet #0 and #1: USDC & WMATIC balances > 0, `checkCanCreate(wallet, WAGER_PARTICIPANT_ROLE) == true`, and allowance ≥ one stake.

### Tests for User Story 1 ⚠️ (write first, ensure they FAIL before T006)

- [X] T004 [P] [US1] Write integration test `test/integration/seed-local.test.js` that, on the in-process Hardhat network, deploys the v2 set, imports and runs the exported `seedLocal()` routine (see T005), then asserts the post-seed invariants for signers 0 and 1: `nativeBalance > 0` (closes FR-003 traceability), `usdc.balanceOf > 0`, `wmatic.balanceOf > 0`, `membershipManager.checkCanCreate(addr, WAGER_PARTICIPANT_ROLE) === true`, `usdc.allowance(addr, wagerRegistry) >= oneStake`. It MUST fail before T005/T006 exist.

### Implementation for User Story 1

- [X] T005 [US1] Create `scripts/operations/seed-local.js`. **Export a reusable `async function seedLocal({ hre, deployment, wallets, amounts })`** that, given resolved contract addresses + signers, performs the mint/grant/approve (so T004 can import and call it without a live node); the script's `main()` resolves the deployment record + signers and calls it, and the file runs `main()` only when invoked directly (`require.main === module`). Behavior: read `deployments/localhost-chain1337-v2.json` (error + non-zero exit with "run deploy:local first" if missing); resolve USDC/WMATIC from `paymentToken`/`wmatic` and WagerRegistry/MembershipManager from `contracts.*`; for signers #0 and #1: `mint` USDC (default 1,000,000) + WMATIC (default 1,000), `grantMembership(addr, WAGER_PARTICIPANT_ROLE, Tier.Bronze, 365)` from the deployer, and `approve(wagerRegistry, large)` for both tokens. Reuse the role hash from `scripts/deploy/lib/constants.js`.
- [X] T006 [US1] Repoint the `seed:local` npm script in `package.json` to `hardhat run scripts/operations/seed-local.js --network localhost` (replacing the broken `scripts/operations/seed-testnet.js` target).
- [X] T007 [US1] Make `scripts/operations/seed-local.js` idempotent and env-parameterizable (`SEED_USDC_AMOUNT`, `SEED_WMATIC_AMOUNT`, `SEED_MEMBERSHIP_DAYS`); guard against running off chainId 1337; print a per-wallet summary (address, balances, membership active?, allowance) and a final "Local environment seeded" line — never print private keys.
- [X] T008 [US1] Run `npx hardhat test test/integration/seed-local.test.js` and confirm the invariants from T004 now pass.

**Checkpoint**: Two wallets are programmatically funded and wager-ready on a local chain — MVP achieved and independently testable without the browser.

## Dependencies & Execution Order

### Phase Dependencies

- **Setup (Phase 1)**: no dependencies — start immediately.
- **Foundational (Phase 2)**: depends on Setup — BLOCKS all user stories.
- **US1 (Phase 3)**: depends on Foundational. This is the MVP.
- **US2 (Phase 4)**: depends on US1 (needs funded wallets) + Foundational (sync target).
- **US3 (Phase 5)**: depends on US1/US2 existing (documents the real commands); docs tasks themselves can be drafted in parallel once the commands are settled.

1. MVP (US1) → funded wallets, automated proof.
2. US2 → `setup:local` orchestrator + app end-to-end create/accept/resolve.
3. US3 → runbook + wallet docs + reset verification.
